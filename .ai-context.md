## 🏗️ Архитектурные решения

### Ключевая концепция: ЕДИНЫЙ PostgreSQL

**Решение**: Все приложения используют ОДИН контейнер PostgreSQL с отдельными базами данных.

**Почему:**
- Экономия ресурсов (RAM, CPU)
- Упрощение управления
- Единая точка для бекапов
- Быстрое добавление новых приложений

**Исключения**: Приложения, которые НЕ поддерживают PostgreSQL (используют SQLite, не требуют БД).

### Текущая структура

```
Traefik (reverse proxy, HTTPS)
    ↓
Nextcloud ──→ PostgreSQL (единый контейнер)
Navidrome ──→ SQLite (встроенная БД)
                ├── nextcloud (database)
                └── [другие БД в будущем]
    ↓
DB Backup Services (по одному на каждую БД PostgreSQL)
    ↓
volumes/backup/pg/nextcloud/
```

## 📝 Переменные окружения (.env)

### Важное правило организации

**КРИТИЧНО:** Переменные окружения должны быть сгруппированы по разделам приложений, а НЕ по типам переменных!

❌ **Неправильно:**
```bash
# Все порты в одном месте
NEXTCLOUD_PORT=8081
NAVIDROME_PORT=4533

# Все домены в одном месте
NEXTCLOUD_DOMAIN=...
NAVIDROME_DOMAIN=...
```

✅ **Правильно:**
```bash
# Nextcloud
NEXTCLOUD_LOCAL_PORT=8081
NEXTCLOUD_DOMAIN=...
POSTGRES_NC_DB=...

# Navidrome
NAVIDROME_LOCAL_PORT=4533
NAVIDROME_DOMAIN=...
```

### Структура
```bash
# Traefik
DOMAIN=ton618.ru
TRAEFIK_DASHBOARD_PORT=8080  # Локальный доступ
TRAEFIK_DASHBOARD_AUTH=...

# PostgreSQL (общие)
POSTGRES_PORT=5432           # Локальный доступ
POSTGRES_ROOT_PASSWORD=...
POSTGRES_USER=postgres

# Nextcloud
NEXTCLOUD_LOCAL_PORT=8081    # Локальный доступ
NEXTCLOUD_DOMAIN=home.ton618.ru
POSTGRES_NC_DB=nextcloud
POSTGRES_NC_USER=nextclouduser
POSTGRES_NC_PASSWORD=...

# Navidrome
NAVIDROME_LOCAL_PORT=4533    # Локальный доступ
NAVIDROME_DOMAIN=music.ton618.ru
NAVIDROME_MUSIC_PATH=/path/to/music

# [Другие приложения добавляются здесь]
```

### Как создаются БД
При первом запуске PostgreSQL выполняется `docker/postgres/init.sql`:
```sql
CREATE DATABASE nextcloud;
CREATE USER nextclouduser WITH ENCRYPTED PASSWORD '...';
ALTER DATABASE nextcloud OWNER TO nextclouduser;
GRANT ALL PRIVILEGES ON DATABASE nextcloud TO nextclouduser;
GRANT ALL ON SCHEMA public TO nextclouduser;
```

## 💾 Система бекапов

### Автоматические бекапы

**Контейнер**: `db-backup` (для Nextcloud)  
**Образ**: `prodrigestivill/postgres-backup-local`  
**Расписание**: Ежедневно в 02:00 (`SCHEDULE: 0 2 * * *`)  
**Хранение**: 7 дней (`BACKUP_KEEP_DAYS: 7`)  
**Папка**: `./volumes/backup/pg/nextcloud/`

### Структура бекапов
```
volumes/backup/pg/
└── nextcloud/
    └── *.sql.gz файлы (автоматически создаются и ротируются)
```

### Восстановление

**Скрипт**: `restore-backup-multi.sh`

**Как работает**:
1. Сканирует `volumes/backup/pg/` и находит папки с БД
2. Показывает список доступных БД и бекапов
3. Использует `case`-блок для определения параметров БД:
   ```bash
   case $DB_NAME in
       "nextcloud")
           DB_USER="${POSTGRES_NC_USER}"        # из .env
           DB_PASSWORD="${POSTGRES_NC_PASSWORD}"
           SERVICE_NAME="nextcloud"              # контейнер приложения
           DB_CONTAINER="db"                     # контейнер PostgreSQL
           ;;
   esac
   ```
4. Останавливает приложение → пересоздаёт БД → восстанавливает → запускает

**Важно**: Для каждой новой БД нужно добавить `case` в скрипт!

## 🌐 Traefik & HTTPS

### Конфигурация

**Файл**: `docker/traefik/traefik.yml` (НЕ в git, содержит email)

**Entry Points**:
- `web` (80) - HTTP, редирект на HTTPS
- `websecure` (443) - HTTPS

**Certificates Resolver**: `letsencrypt`
- Email: `f0x@ton618.ru`
- Challenge: `httpChallenge` (порт 80)
- Storage: `/acme.json`

### SSL сертификаты

**Файл**: `volumes/traefik/acme.json`  
**Права**: 600 (КРИТИЧНО!)  
**Содержит**: Приватные ключи и сертификаты Let's Encrypt

**Проблема**: При первом запуске файл может не существовать или иметь неправильные права
**Решение**: 
```bash
touch volumes/traefik/acme.json
chmod 600 volumes/traefik/acme.json
docker compose restart traefik
```

### Важное о порте 80

- Порт 80 **ОБЯЗАТЕЛЕН** для HTTP Challenge (Let's Encrypt)
- Если порт 80 занят (например, Apache2) - нужно остановить конфликтующий сервис
- Проверка: `sudo lsof -i :80`

### Процесс для приложений с PostgreSQL (4 шага)

**1. Добавить в `.env`:**
```bash
NEWAPP_DB=newapp
NEWAPP_DB_USER=newappuser
NEWAPP_DB_PASSWORD=secure-password
```

**2. Обновить `docker/postgres/init.sql`:**
```sql
CREATE DATABASE newapp;
CREATE USER newappuser WITH ENCRYPTED PASSWORD 'password';
ALTER DATABASE newapp OWNER TO newappuser;
GRANT ALL PRIVILEGES ON DATABASE newapp TO newappuser;
GRANT ALL ON SCHEMA public TO newappuser;
```

**3. Добавить в `docker-compose.yml`:**
```yaml
newapp:
  image: newapp-image
  depends_on: [db]
  environment:
    DB_HOST: db
    DB_NAME: ${NEWAPP_DB}
    DB_USER: ${NEWAPP_DB_USER}
    DB_PASSWORD: ${NEWAPP_DB_PASSWORD}
  labels:
    - "traefik.enable=true"
    - "traefik.http.routers.newapp.rule=Host(`newapp.${DOMAIN}`)"
    - "traefik.http.routers.newapp.entrypoints=websecure"
    - "traefik.http.routers.newapp.tls.certresolver=letsencrypt"

newapp-backup:
  image: prodrigestivill/postgres-backup-local
  environment:
    POSTGRES_HOST: db
    POSTGRES_USER: ${POSTGRES_USER}
    POSTGRES_PASSWORD: ${POSTGRES_ROOT_PASSWORD}
    POSTGRES_DB: ${NEWAPP_DB}
    SCHEDULE: "0 2 * * *"
    BACKUP_KEEP_DAYS: 7
  volumes:
    - ./volumes/backup/pg/newapp:/backups
```

**4. Добавить в `restore-backup-multi.sh`:**
```bash
"newapp")
    DB_USER="${NEWAPP_DB_USER}"
    DB_PASSWORD="${NEWAPP_DB_PASSWORD}"
    SERVICE_NAME="newapp"
    DB_CONTAINER="db"
    ;;
```

### При работе с документацией

**Принципы проекта:**
1. **Единый PostgreSQL** - это архитектурное решение, не предлагать отдельные контейнеры БД

3. **Фиксированные версии Docker образов** - НИКОГДА не использовать тег `latest`, только конкретные версии (например `5.1.4-2`, `0.60.3`)
4. **Простота** - документация должна быть понятной
5. **Безопасность** - не хранить секреты в git

**Документы:**
- `README.md` - общий обзор, быстрый старт
- `docs/ADDING_DATABASES.md` - конкретные примеры добавления приложений
- `docs/ARCHITECTURE.md` - подробная архитектура, диаграммы
- `docs/TRAEFIK_SETUP.md` - настройка HTTPS и доменов

### При обсуждении бекапов

**Скрипт `restore-backup-multi.sh`:**
- Определяет БД по **структуре папок** `volumes/backup/pg/`
- Параметры БД задаются через **case-блок** (жёсткая привязка)

- `${RED}`, `${GREEN}`, `${YELLOW}`, `${BLUE}` - ANSI escape-коды
